# PRD-000013-BUG

## Short Name

Fix Render Stop After Table

## Goal

Identify and fix the markdown rendering failure that causes Quill to render a broken or partial table and then stop rendering the rest of the document.

## Context

The issue has been observed while rendering `C:\Projects\Work\MarketResearch\SKILL-BACKLOG.md`. In the render pane, Quill reaches the text `**If paywalled, extract maximum value from public surfaces before moving on:**`, begins rendering the following table incorrectly, and then stops rendering the remainder of the markdown output. The underlying cause is currently unknown, so the first step is to reproduce it reliably and isolate whether the failure is caused by the table parser, surrounding markdown structure, or a broader render-pipeline exception.

## Scope

In:

- reproduce the failure against `C:\Projects\Work\MarketResearch\SKILL-BACKLOG.md`
- identify the exact markdown pattern or parser state that produces the broken partial table and render cutoff
- fix the rendering behavior so the full document continues rendering after the table
- add or update targeted regression coverage if the markdown pipeline has a suitable test seam
- record any notable parser limitations or edge-case handling decisions discovered during the fix

Out:

- unrelated markdown rendering changes not needed for this bug
- broad editor or preview redesign work
- speculative parser rewrites without a narrowed root cause

## Proposed Behavior

- The full document should continue rendering after the affected table instead of halting at the first broken row.
- Inline code and punctuation inside the `How` column should remain visible as normal table-cell content rather than expanding into oversized orphaned fragments.
- The first affected row should stay aligned as a three-column table row under `Surface`, `What to extract`, and `How`.

## Plan

- Reproduce the failure in the smallest parser or render harness that can use the same markdown table path as Quill.
- Reduce the problematic table row to the smallest markdown fragment that still causes the `How` column to fragment and the render output to stop.
- Identify whether the break happens during table row splitting, inline-code handling, HTML generation, or downstream render composition.
- Capture the smallest safe fix and the regression checks needed before moving to `02 - Implement`.

## Acceptance Criteria

- The table that follows `**If paywalled, extract maximum value from public surfaces before moving on:**` renders as a normal three-column table without oversized orphaned fragments in the `How` column.
- Rendering continues past the affected table so the remainder of `C:\Projects\Work\MarketResearch\SKILL-BACKLOG.md` still appears in the render pane.
- The markdown content that currently appears to fragment into large code-styled text in the screenshot remains contained inside the intended table cell content after the fix.
- The final fix is verified against the original repro file and against the smallest reduced markdown fragment that reproduces the failure during planning.

## Implementation Notes

- Treat this as a table-cell rendering bug in the live preview pipeline, not as a generic paragraph or heading rendering change.
- Escape raw table-cell content before applying inline formatting so HTML-like text inside backticks stays text in the rendered cell instead of becoming real tags.
- Support variable-length inline code fences inside table cells so both single-backtick and double-backtick spans render correctly there.
- Keep unmatched backtick fences literal rather than manufacturing a broken `<code>` span.
- Bump the versioned `markdown.js` asset URL in `code/quill.html` so browser-visible preview sessions reload the updated parser.

## Verification

- Verified `code/scripts/markdown.js` passes `node --check`.
- Verified in a Node-based live-preview harness that table-cell rendering now keeps HTML-like text inside backticked content as visible text rather than turning it into live tags.
- Verified in the same harness that rendering continues past the affected table and preserves the trailing paragraph after the previously broken section.
- Manually verified the fix in two browsers against Quill's real render pane.
- Added repository-backed screenshot evidence showing the corrected table rendering and preserved code-styled cell content.
