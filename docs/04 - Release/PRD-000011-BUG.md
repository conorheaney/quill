# PRD-000011-BUG

## Short Name

Fix Fenced Markdown Load State

## Goal

Make Quill fully load Markdown files that contain fenced code blocks so the editor, render pane, and outline stay in sync and the app does not show a false file-open failure.

## Context

Quill can currently enter a corrupted half-loaded state when a Markdown file contains fenced code blocks. The source editor updates, but preview rendering throws before the render pane and outline can refresh, and the load flow surfaces a misleading failure toast after partially applying the file content.

## Scope

In:

- reproduce the fenced-code-block load failure against a real Markdown sample
- fix the shared Markdown parsing path so code-fence language normalization is available inside the markdown module
- verify that loading a fenced-markdown document no longer leaves Quill in a split state

Out:

- unrelated Markdown feature expansion
- broader parser redesign
- recent-files UX changes not required for this bug

## Proposed Behavior

- Opening a Markdown file with fenced code blocks should update the source, render pane, and outline together.
- The file-open flow should only show a failure toast when the file truly cannot be opened or parsed.
- Shared Markdown parsing helpers should not depend on private functions scoped to other modules.

## Acceptance Criteria

- Parsing Markdown with fenced code blocks no longer throws due to missing language-normalization helpers.
- The supplied `SKILL.md` sample can pass through the shared Markdown parser without error.
- Quill no longer enters a half-loaded state for fenced-markdown files.

## Implementation Notes

- Keep the fix local to the shared markdown module unless verification shows a second fault in the load flow.
- Reuse the existing language normalization mapping so preview editing and file-open parsing stay aligned.
- Bump the versioned `markdown.js` asset URL in `code/quill.html` so direct-browser Chrome sessions pick up the parser fix reliably.

## Next Step

No further implementation work is required for this item.
