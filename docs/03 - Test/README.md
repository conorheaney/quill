# Test Stage

Use this folder for backlog items that are ready to verify and need explicit checking notes.

Typical content:

- test checklist
- smoke-test notes
- known failures
- decision on whether the item is ready to release

Rules:

- keep this folder small
- prefer one item in test at a time
- move the file to `docs/04 - Release/` after acceptance, or back to `docs/02 - Implement/` if more work is needed
