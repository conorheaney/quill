# Plan Stage

Use this folder for backlog items that need more planning detail than fits in `docs/BACKLOG.md`.

Typical content:

- scope notes
- acceptance criteria
- sequencing decisions
- risks worth resolving before implementation

Rules:

- keep one file per backlog item when needed
- name the file exactly after the backlog ID, for example `PRD-0002-TECH.md`
- move the file to `docs/02 - Implement/` when implementation starts
